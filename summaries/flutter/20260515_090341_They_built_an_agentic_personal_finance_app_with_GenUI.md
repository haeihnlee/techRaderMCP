# They built an agentic personal finance app with GenUI

- **컨퍼런스**: Flutter
- **출처**: https://www.youtube.com/watch?v=QCElXNUr3RA
- **요약 일시**: 2026-05-15 09:03:41

---

## 🔑 핵심 요약
- **Flutter** + **GenUI(Generative UI)** 조합으로 LLM이 직접 UI를 그려내는 에이전트형 금융 시뮬레이터 데모 공개
- **Gemini**에게 디자인 시스템 컴포넌트 카탈로그를 제공하면, 텍스트 대신 라디오 버튼·카드·그래프 등 **UI로 응답**
- Flutter 단일 코드베이스로 **Wasm 웹 / Android / iOS / 데스크톱** 모두에서 동일하게 동작

---

## 📣 주요 발표 내용
- **GenUI 개념**: LLM이 자연어 텍스트가 아닌 **UI 컴포넌트로 대화**하는 새로운 패러다임
- **Catalog 메커니즘**: 개발자가 `catalog`에 디자인 시스템 컴포넌트 + 사용 방법을 정의 → Gemini가 그 안에서만 UI 렌더링
- 데모 앱(Personal Finance Simulator) 흐름:
  - 초기 2개 화면은 **정적 Flutter 화면** (대화 진입점)
  - 이후부터는 **Gemini가 동적으로 UI 생성** (라디오 버튼, 입력 필드, 대시보드 등)
  - 사용자 입력 → Gemini → 다음 단계 UI 생성 반복
- **시스템 프롬프트**로 "사용자의 재무 상황을 묻고, 충분한 정보가 모이면 대시보드를 제시하라"는 지시만 제공
- Gemini가 **그래프·카드 컴포넌트**까지 자율적으로 선택해 결과 대시보드 구성 (예: FIRE/FAT FIRE 목표 대비 진척도)
- Flutter `Wasm` 컴파일 덕분에 Chrome에서도 네이티브 수준 경험

---

## 💡 개발자 포인트
- **브랜드 일관성 유지**: 디자인 시스템을 카탈로그로 제공하므로 LLM이 만들어도 **브랜드 가이드 안에서만** UI 생성 → 브랜드 안전(brand-safe)
- 대화형 앱을 **텍스트 chat UI 없이** 구현 가능 → 더 풍부한 시각적·앱다운 UX
- 정적 화면과 GenUI 동적 화면을 **혼합**하는 패턴이 실용적 (진입점은 정적, 본 흐름은 동적)
- 크로스플랫폼 1개 코드베이스로 모바일·웹(Wasm)·데스크톱 모두 커버

> ⚠️ **주의**: 데모 수준의 사례이므로, 프로덕션 적용 시 LLM의 UI 생성 결과에 대한 **검증·폴백 전략**과 **레이턴시·비용** 고려가 필요합니다.
