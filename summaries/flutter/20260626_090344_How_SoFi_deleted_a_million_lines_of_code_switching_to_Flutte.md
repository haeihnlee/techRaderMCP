# How SoFi deleted a million lines of code switching to Flutter

- **컨퍼런스**: Flutter
- **출처**: https://www.youtube.com/watch?v=RBObC8T7ud0
- **요약 일시**: 2026-06-26 09:03:44

---

## 🔑 핵심 요약
- 핀테크 **SoFi**(회원 약 1,500만)가 네이티브(iOS/Android) 앱을 **Flutter**로 마이그레이션한 사례
- 마이그레이션 도중 전체 코드 **100만 줄 삭제**, 기능 단위 비교 시 **약 60% 코드 감소**
- **in-place rewrite**(점진적 재작성) + **feature flag**로 보안·안정성을 유지하며 단계적 전환

---

## 📣 주요 발표 내용
- **마이그레이션 방식**: 네이티브 앱 안에 독립 **Flutter 모듈**을 두고, 기능을 하나씩 Flutter로 이전
- 기능이 Flutter로 작성되면 **feature flag**로 켜고, 대응되는 네이티브 기능을 점진적으로 종료
- **Example app(예시 앱)** 활용으로 생산성 향상
  - 팀별 담당 기능을 **package** 단위로 개발·테스트
  - CI에서 해당 **package만 타겟 빌드/테스트**, **스크린샷 테스트**까지 로컬에서 수행
- 개발 → 코드 리뷰 → 빌드까지의 **이터레이션 시간 대폭 단축**
- SoFi는 **Flutter 코드베이스 컨트리뷰터**로 참여(약 150명, 연내 200명 돌파 예상)

---

## 💡 개발자 포인트
- **"two of everything" 문제 해소**: 네이티브 전용 앱은 iOS/Android 두 벌을 유지 → 디자인·A/B 테스트·구현 파이프라인이 모두 이중화되어 작업 기간이 2배

> 금융 앱 특성상 **모든 기기에서 동일한 UX**가 핵심 — 회원 신뢰 확보 + 고객 지원 단순화(OS·버전별 분기 제거)를 위해 일관된 화면이 필수.

- **Mobile core 팀 역할 변화**: 기존엔 크래시 디버깅에 시간 소모 → 마이그레이션 후 **조직 전체 생산성을 높이는 도구·시스템 구축**으로 전환
- 컴포넌트 일괄 수정부터 **모션·애니메이션 단위 디테일**까지, 필요한 모든 수준에서 유연하게 작업 가능
- 점진적 마이그레이션 전략(모듈 + feature flag)은 **무중단·저위험** 전환을 노리는 대규모 앱에 참고할 만한 패턴
