## 🔑 핵심 요약
- **A2UI(Agent-to-UI)** 는 에이전트가 채팅 사이드바를 넘어 **앱의 실제 UI를 생성·조작**할 수 있게 해주는 오픈 프로토콜
- 통신은 **JSON 블록**으로 이뤄져 보안·속도·이식성을 모두 확보 (실행 코드 없음, LLM 친화적, 모든 언어 파싱 가능)
- Flutter, React, Angular, Lit 렌더러 제공 + **Kotlin/Swift 작업 중**, Python `a2ui-agent` 라이브러리로 에이전트 측 구현 가능

---

## 📣 주요 발표 내용
- **GenUI 정의**: 에이전트가 텍스트 답변만이 아니라 **UI 컴포넌트의 구성·상태를 자신의 컨텍스트로 인식**하고 생성·상호작용까지 수행
- **A2UI 프로토콜**은 에이전트와 클라이언트가 UI 구성, 데이터, 이벤트를 주고받기 위한 표준 — 대화 스트림 안에 JSON 블록으로 삽입
- **3계층 에코시스템**:
  - Protocol layer (A2UI 스펙)
  - Agent libraries (`a2ui-agent` for Python, ADK 검증)
  - Client/Framework adapters (Flutter, React, Angular, Lit 등)
- **6가지 빌딩 블록** (Flutter 기준):
  - `Transport` — 에이전트 토큰 스트림에서 A2UI 메시지를 분리·전달
  - `Catalog` — 에이전트가 사용 가능한 UI 컴포넌트 정의 집합
  - `Data model` — UI를 뒷받침하는 Observable 중앙 데이터 스토어
  - `Surface` — 생성된 UI가 렌더링되는 앱 내 지정 위치
  - `SurfaceController` — Surface의 생성/소멸/업데이트 관리
  - `Conversation` — Transport + SurfaceController를 묶은 단일 API
- **양방향 협업**: 사용자가 체크박스를 탭하면 에이전트가 그 상태를 인지하고, 에이전트가 UI를 수정해도 사용자가 즉시 시각화 가능 (Todo 앱 코드랩 데모)
- 데모에서 텍스트 전용 클라이밍 챗봇을 **카드·버튼이 포함된 동적 UI**로 전환 — 패키지 통합 + 빌딩 블록 연결만으로 구현

## 💡 개발자 포인트
- **Bring-your-own-model 아키텍처**: `AgentTransportAdapter`의 `sendRequest()` / `addChunk()` 만 구현하면 어떤 LLM 공급자든 연결 가능 (데모는 Dart `dartantic` 사용)
- **시스템 프롬프트가 핵심**: 에이전트에게 "어떤 상황에서 어떤 컴포넌트를 쓸지" 가이드하는 **prompt fragment**를 반드시 제공해야 함
- **Catalog 커스터마이즈**: 기본 카탈로그에서 불필요한 컴포넌트(`image`, `video`, `audioPlayer` 등) 제거 또는 자체 컴포넌트 추가 가능

> ⚠️ A2UI는 현재 `0.9` 버전 (pre-1.0) — 프로토콜·API가 변경될 수 있으므로 프로덕션 도입 시 버전 고정 권장

> 💡 GenUI의 진짜 가치는 **정적 화면 라우팅의 한계 극복** — 사전에 설계된 화면 경로를 따라가지 않고, 사용자의 컨텍스트에 맞춰 **단일 화면에 필요한 컴포넌트를 그때그때 조립**
