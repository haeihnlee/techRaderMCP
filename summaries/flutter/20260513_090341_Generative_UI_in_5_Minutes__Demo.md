## 🔑 핵심 요약
- **Generative UI (Genuine)** 는 LLM 에이전트와의 "Wall of Text" 문제를 해결하기 위해, 응답 UI 자체를 LLM이 동적으로 결정하게 하는 패러다임이다
- **Gemini**가 질문 내용·선택지·UI 컴포넌트 종류(버튼/콤보박스/슬라이더 등)를 모두 실시간 결정
- 사용자 입력을 정형화된 위젯으로 받으므로 **오타·잘못된 답변** 없이 LLM에 깨끗한 데이터가 전달됨

---

## 📣 주요 발표 내용
- 기존 에이전트 대화의 문제: 긴 텍스트 단락으로 질문 → 사용자는 오타 섞인 자유 텍스트로 답변 → LLM 혼란
- **Generative UI** 데모: "달에서 토끼 생일 파티 기획" 시나리오로 동적 UI 생성 시연
- LLM이 상황에 맞춰 선택하는 UI 컴포넌트:
  - **2D 슬라이더** (다차원 분위기 선택: high-energy hop ↔ low-gravity chill, intimate burrow ↔ galactic gala)
  - **콤보박스** (단일 선택 질문)
  - **버블/멀티 셀렉트** (여러 항목 선택: 우주 간식 종류)
  - **일반 슬라이더** (강도 조절: 연회 규모)
- 초대받은 게스트 화면에서도 메타데이터가 모두 **위젯 형태**로 표현됨
- 화면 우측 상단에 **Gemini API 호출 + UI 구성 소요 시간**을 표시하는 빨간 숫자 인디케이터 포함

---

## 💡 개발자 포인트
> **핵심 패러다임 전환**: UI를 사전에 정의하는 것이 아니라, 프롬프트 컨텍스트에 따라 **LLM이 런타임에 UI를 생성**하도록 위임한다.

- 사용자 응답이 항상 구조화된 형식으로 들어오므로 **LLM 다음 호출의 입력 품질**이 크게 향상됨
- 자유 텍스트 입력보다 **비즈니스 로직 제약**(슬라이더 범위, 단일/다중 선택 등)을 LLM이 함께 결정 가능
- Flutter의 **위젯 시스템**이 동적 UI 구성에 적합 — LLM이 위젯 타입을 선택하고 앱이 렌더링
- 적용 사례: 폼 입력, 설정 마법사, 에이전트 대화 등 **선택지가 많은 인터랙션** 전반
- 응답 시간(LLM 호출 + UI 빌드)이 UX의 핵심 변수이므로 모니터링 필수
