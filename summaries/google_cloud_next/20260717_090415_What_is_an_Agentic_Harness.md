# What is an Agentic Harness?

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=W9BX0jyzd2k
- **요약 일시**: 2026-07-17 09:04:15

---

## 🔑 핵심 요약
- **에이전트**란 "**LLM + 도구**가 목표 달성을 위해 **루프(loop)**로 실행되는 것"이라는 Simon Willison의 정의를 인용해 설명함
- **에이전틱 하네스(agentic harness)**는 LLM 자체가 아니라 "LLM 이후의 모든 것" — 도구 제공, 루프 실행, 결과물 평가 로직 등 LLM을 에이전트로 만드는 인프라 전체를 의미
- `Claude Code`, `Antigravity`, `Codex` 같은 도구는 하네스와 인터페이스(UI)가 결합된 형태지만, 개념적으로는 **하네스와 인터페이스를 분리**해서 생각하는 것이 핵심

---

## 📣 주요 발표 내용
- LLM의 본질은 **다음 토큰 예측**(next token prediction)이며, 입력 텍스트를 기반으로 이어질 내용을 생성하는 "정교한 자동완성"에 가까움
- **function calling**의 등장으로 LLM에 "이런 함수를 호출할 수 있다"는 시그니처(파라미터 포함)를 프라이밍할 수 있게 됨
- 함수/도구의 핵심 역할은 **LLM 학습 데이터에 없는 외부 컨텍스트를 가져와** 대화 컨텍스트에 주입, 응답 품질을 보강하는 것
- 하네스의 "루프" 구성 요소는 단순 1회 응답이 아니라, **프로그램적으로 LLM 출력을 평가**해 목표 달성 여부(done/not done)를 판단하고, 미완료 시 추가 지시를 다시 LLM에 주입하는 반복 구조
- 하네스 = **도구 제공 로직 + 루프 실행 로직 + 결과 평가 로직**을 아우르는 인프라 (필요 인프라 양이 크지 않아도 됨)

---

## 💡 개발자 포인트
- 하네스를 IDE나 "에이전트를 실행하는 무언가" 정도로 오해하기 쉬우나, 실제로는 **UI보다 더 낮은 레벨의 개념** — LLM의 로직·행동을 제어하는 코드 그 자체
> **하네스와 인터페이스는 분리 가능한 개념**입니다. 동일한 에이전틱 하네스 위에 콘솔 UI, 채팅 UI, 혹은 UI 없는 순수 프로그래밍 인터페이스(자율 에이전트)를 자유롭게 교체해 얹을 수 있습니다.
- 사용 사례에 따라 **사용자 인터페이스 자체가 필요 없는 자율 에이전트**(순수 프로그래밍 인터페이스)를 구성할 수도 있음 — 코딩 에이전트와 소비자용 챗 앱이 인터페이스는 다르지만 동일한 하네스로 동작할 수 있음을 시사
