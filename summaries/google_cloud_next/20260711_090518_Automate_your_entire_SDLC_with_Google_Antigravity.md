# Automate your entire SDLC with Google Antigravity

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=K3YYr6yauAw
- **요약 일시**: 2026-07-11 09:05:18

---

## 🔑 핵심 요약
- Google **Antigravity**(에이전틱 코딩 도구)로 SDLC 전체(이슈 파악 → 설계 → 구현 → 리뷰/테스트)를 자동화하는 방법을 실제 오픈소스 PR 사례로 시연
- 코드 생성에만 에이전트를 쓰고 **유지보수·리뷰·테스트에는 충분히 활용하지 않는 것**이 버그·기술부채가 쌓이는 주된 원인이라고 지적
- 복잡한 이슈/디스커션 요약, PR 파일별 변경사항 요약 등 **자연어 대화로 SDLC 전 단계를 보조**하는 워크플로우를 소개

---

## 📣 주요 발표 내용
- **이슈 파악**: 복잡한 오픈소스 이슈와 첨부된 디스커션을 직접 읽는 대신, 에이전트에게 "이 이슈와 디스커션을 요약해줘, 현재 상태가 뭐야?"라고 요청해 빠르게 파악
- **설계**: GitHub 프로필 점수 사이트에 새 기능(전체 사용자 대비 순위 분포 탭)을 추가하며, 에이전트와 대화로 "2만 개 계정을 프라이버시 친화적으로 가져와 저장하는 방법"을 설계
- **구현**: 작업할 저장소(포크) URL을 알려주고 "이 브랜치를 유지한 채 clone/pull 하고, 국가 선택기(country selector)와 동일한 방식으로 이름도 오버레이 편집 후 URL 파라미터에 저장하는 기능을 만들어줘"처럼 자연어 프롬프트로 기능 구현을 지시
- **리뷰/테스트**: 완성된 PR의 변경 파일이 많을 때, 에이전트에게 "파일별로 1~2문장씩 요약해줘"라고 요청해 파일 단위로 빠르게 리뷰하고 필요한 부분만 깊게 파고드는 방식 시연

---

## 💡 개발자 포인트
- 코드 작성뿐 아니라 **이슈 요약 → 설계 논의 → 구현 지시 → PR 리뷰**까지 SDLC 전 구간에서 에이전트를 대화형으로 활용하면 반복 작업 시간을 크게 줄일 수 있음
- 이름 파싱처럼 "일부 케이스만 맞는" 하드코딩된 로직보다는, **사용자가 직접 커스터마이징 가능한 UI(오버레이 편집 + URL 파라미터 저장)**로 설계하는 편이 더 견고한 해결책이 될 수 있음

> 발표자는 "에이전트에게 코드 생성만 과도하게 의존하고, 유지보수·리뷰·테스트에는 충분히 활용하지 않는 것"이 버그와 기술부채가 늘어나는 핵심 원인이라고 강조함
