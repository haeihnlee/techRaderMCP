## 핵심 요약 (3줄)
- AI 에이전트 프로토타입이 로컬에서는 잘 동작하지만 스케일링 시 실패하는 문제를 라이브로 진단하고 수정하는 세션
- 'Project Titanium'이라는 아웃리치 에이전트의 하드코딩 병목점과 정확도 리스크를 60분 안에 실시간으로 해결
- "vibe-coding" 방식으로 빠르게 만든 AI 프로토타입의 실제 운영 환경 전환 시 발생하는 공통 문제를 다룸

## 주요 발표 내용
- **The Agent Clinic** 포맷: 실제 개발자가 제출한 문제 있는 AI 에이전트를 전문가와 함께 라이브로 수정하는 세션
- **진단 대상**: Jacob Badish의 아웃리치 에이전트 'Project Titanium'
- **주요 문제점**:
  - 하드코딩된 병목(Hardcoded Bottlenecks) 존재
  - 정확도 리스크(Accuracy Risks) 내포
  - 로컬 환경과 스케일링 환경 간의 동작 불일치
- **진행 방식**: Host Luis Sala(Lead Diagnostician)와 제출자 Jacob이 60분 제한 시간 내 실시간 수정

## 개발자에게 중요한 포인트
- **"Vibe-coding" 함정 주의**: 빠른 프로토타이핑으로 만든 AI 에이전트는 프로덕션 환경에서 쉽게 깨질 수 있음
- **하드코딩 지양**: 에이전트 내 하드코딩된 값/로직은 스케일링 시 치명적인 병목이 됨
- **정확도 검증 필요**: 프로토타입 단계에서 넘어가기 전 정확도 리스크를 반드시 점검해야 함
- **실전 디버깅 참고**: 실제 에이전트 코드를 라이브로 수정하는 과정을 통해 AI 에이전트 개발 모범 사례를 학습 가능
