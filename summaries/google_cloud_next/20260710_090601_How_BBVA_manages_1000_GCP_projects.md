# How BBVA manages 1,000+ GCP projects

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=SKT9wiBh29A
- **요약 일시**: 2026-07-10 09:06:01

---

## 🔑 핵심 요약
- 스페인 금융사 **BBVA**(고객 8,100만+, 25개국)가 **15년간** Google Cloud를 사용하며 **수천 개의 GCP 프로젝트**와 수백 개의 애플리케이션을 운영 중
- 초기 `App Engine` 기반 내부 개발에서 시작해, 현재는 **Terraform** 기반 IaC 자동화 파이프라인으로 프로비저닝·운영을 자동화
- **Cloud Hub / App Hub**로 프로젝트 단위로 흩어진 리소스를 **애플리케이션 단위 뷰**로 묶어 운영 가시성을 확보
- 다음 단계로 **에이전틱(Agentic) AI 레이어**를 도입해 콘솔 점검 없이 이상 발생 시 알림을 받는 자동 운영을 계획

---

## 📣 주요 발표 내용
- **아키텍처 패턴**: 애플리케이션을 작은 컴포넌트로 분해하고, 각 컴포넌트를 **하나의 GCP 프로젝트**에 매핑하는 구조
- 프로젝트 수가 늘면서 서비스 매니저가 운영 데이터를 모으기 어려워지는 **스케일 문제** 발생 → 수동 운영(프로비저닝·모니터링·배포)에서 자체 자동화 도구 개발로 전환
- **App Hub**: 애플리케이션의 경계를 정의하고 여러 프로젝트의 리소스를 하나의 애플리케이션 뷰로 집계
- **Cloud Hub**: 이슈·알림·**비용 최적화** 정보를 단일 대시보드에서 확인 — 프로젝트 생성 시 자동으로 애플리케이션에 연결되도록 파일럿 진행 중
- **AI 계획**: 에이전트가 정기 점검을 대신 수행하고, 설정에 문제가 있을 때만 이메일/알림을 보내는 운영 자동화 구상

---

## 💡 개발자 포인트
- 컴포넌트당 GCP 프로젝트 1개 패턴은 격리에는 유리하지만, 규모가 커지면 **애플리케이션 단위 관측/운영 도구**(App Hub·Cloud Hub)가 필수
- **자동화를 처음부터 설계하라** — BBVA는 초기에 수동 운영으로 시작했다가 뒤늦게 자동화 도구를 자체 개발해야 했음. 프로비저닝·운영 양쪽 모두 `Terraform` 등 IaC로 시작할 것을 권장

> **핵심 교훈**: 보안(Security)과 복원력(Resiliency)은 나중에 추가하지 말고 **설계 단계부터** 포함시킬 것. "Resiliency ready"한 설계를 처음부터 목표로 해야 함
