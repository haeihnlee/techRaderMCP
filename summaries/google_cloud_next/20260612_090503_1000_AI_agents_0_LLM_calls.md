# 1,000 AI agents, 0 LLM calls?

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=EtxJJJ_7WQY
- **요약 일시**: 2026-06-12 09:05:03

---

## 🔑 핵심 요약
- **1,000개의 AI 에이전트**가 마라톤을 달리는 데모 — 레이스 시작 후에는 대부분 **LLM을 전혀 호출하지 않음**
- 핵심 원칙: **추론(reasoning)은 LLM에게, 예측 가능한 로직은 일반 코드로** 처리
- **ADK**의 `before_model_callback`으로 모델 호출 전에 개입해 **결정적(deterministic) 코드**를 대신 실행, 비용 절감

---

## 📣 주요 발표 내용
- **ADK(Agent Development Kit)**의 **콜백(callback)** 기능으로 에이전트 라이프사이클의 각 단계를 가로챌(intercept) 수 있음
- `before_model_callback`을 사용하면 에이전트가 모델을 호출하기 **직전에 실행을 중단**하고 일반 코드를 대신 실행 가능
- 1,000개 에이전트 규모에서도 LLM 호출을 0에 가깝게 줄여 **대규모 멀티 에이전트 시뮬레이션**을 저비용으로 운영하는 패턴 시연

---

## 💡 개발자 포인트
- 에이전트를 설계할 때 "**언제 LLM이 실제로 실행되어야 하는가**"를 먼저 판단할 것 — 모든 스텝에 LLM이 필요한 것은 아님
- 로직이 예측 가능하다면(규칙 기반 이동, 상태 업데이트 등) `before_model_callback`에서 **결정적 코드로 대체**해 토큰 비용과 지연 시간을 동시에 절감
> 멀티 에이전트 시스템의 비용 폭증은 대부분 불필요한 LLM 호출에서 발생합니다. 콜백으로 라이프사이클을 가로채는 패턴을 기본 설계로 고려하세요.
