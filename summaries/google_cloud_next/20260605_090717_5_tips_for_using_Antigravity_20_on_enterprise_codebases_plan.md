## 🔑 핵심 요약
- **Antigravity 2.0**를 실제 **엔터프라이즈 코드베이스**에서 활용하는 방법을 다룬 2부작 중 **1부(계획 단계)** 영상이다.
- AI 에이전트를 단순 텍스트 입력창이 아니라 **"디지털 인턴(Ducky)"**으로 대하고, 코드를 건드리기 전 **충분히 브리핑**하는 것이 핵심.
- 멀티 레포 연결, 규칙 계층화, 가드레일 설정, 음성 입력 기획, `{slash} grill me` 역질문까지 **5가지 사전 준비 팁**을 제시한다.

---

## 📣 주요 발표 내용

**Tip 1 — 레포 경계 허물기 (Break the repo walls)**
- 엔터프라이즈 앱은 보통 **프론트엔드 / 백엔드 / 공유 패키지**가 여러 레포로 흩어져 있음
- Antigravity 2.0는 **여러 레포를 하나의 프로젝트**로 묶어 **크로스 레포 변경**을 지원
- 예: 백엔드 데이터 모델 수정 → 프론트엔드 `TypeScript` 인터페이스까지 한 번에 동기화

**Tip 2 — 규칙의 계층 구조 (Hierarchy of rules)**

| 레벨 | 위치 | 용도 |
|------|------|------|
| **Global** | 홈 디렉터리 `~/.gemini/gemini.md` | 모든 앱 공통 규칙 (예: 최대 라인 길이 80자) |
| **Project** | 프로젝트 루트 `.agents/rules` | 해당 프로젝트/앱별 코딩 표준 |
| **README** | 개별 폴더 내 `readme.md` | 하이퍼 로컬 컨텍스트 (예: 백엔드 통합 테스트 실행법) |

**Tip 3 — 실수의 폭발 반경 차단 (Contain the blast radius)**
- 에이전트 설정에서 **custom security preset** 선택
- 터미널 명령 실행 전 **항상 리뷰 요구**, 자주 쓰는 명령은 실행 시 **allow list**에 점진적으로 추가
- **Sandbox mode**(프리뷰) 활성화 → OS 레벨 격리로 파괴적 셸 작업·무단 네트워크 호출 차단
- 클라우드 단에서 개발 머신이 **프로덕션 시스템에 접근 못 하도록** 권한 분리

**Tip 4 — 음성 입력으로 기획 (Native voice input)**
- 복잡한 아키텍처/요구사항은 **타이핑 대신 음성**으로 자유롭게 구술
- 프린시플 엔지니어와 대화하듯 레거시 quirk까지 messy하게 풀어내며 모델과 왕복 → 의도를 명확화

**Tip 5 — 에이전트가 나를 심문하게 하라 (`{slash} grill me`)**
- 음성 프롬프트 후 `{slash} grill me` 명령 입력
- Ducky가 코드베이스를 보고 **모호한 요구사항·결정 필요 항목에 대해 5~6개 날카로운 질문**을 역으로 던짐
- 사전에 답하면 나중에 코드 재작성 비용을 크게 절감

## 💡 개발자 포인트
- AI 에이전트를 **"멘토링이 필요한 인턴"**으로 프레이밍하면 경계 설정·코칭 관점이 생겨 결과 품질이 올라간다.

> ⚠️ **가드레일은 선택이 아니라 필수.** Sandbox mode와 터미널 명령 리뷰를 켜두지 않으면 에이전트의 자동 루프가 머신에서 통제 불능으로 실행될 수 있다. 특히 **개발 머신의 프로덕션 접근 권한은 반드시 차단**할 것 — 도구 설정만으로는 부족하고 클라우드 권한 레벨에서 끊어야 한다.

- 규칙은 **Global → Project → README** 3계층으로 분리하면 컨텍스트 관리가 쉬워지고, 폴더별 `readme.md`는 동료에게도 문서로서 도움이 된다.
- 코드 작성 전 **음성 기획 + `grill me` 역질문**으로 요구사항을 충분히 다지는 것이 "처음부터 5분 만에 앱 만들기" 류 데모보다 실무에 훨씬 유용하다.
