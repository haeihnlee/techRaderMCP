# Building with ADK & A2A: How multi-agent collaboration designs a game

## 🔑 핵심 요약
- **ADK**(Agent Development Kit)와 **A2A**(Agent-to-Agent) 프로토콜로 **여러 전문 에이전트가 협업**해 5분 만에 비디오 게임 기획안을 완성하는 라이브 데모
- 각 에이전트는 **게임플레이 / 아트 디렉션 / 프로덕션 / 마케팅**이라는 단일 역할에 집중하고, 중앙 **오케스트레이션 에이전트**가 결과를 하나의 제안서로 통합
- 에이전트 간 결정이 **충돌하면 사람(human-in-the-loop)** 에게 최종 판단을 넘기는 흐름을 강조

---

## 📣 주요 발표 내용
- **역할 분담형 멀티 에이전트** 구조: 각 태블릿(스테이션)이 하나의 에이전트이며 자기 역할에 따라 선택을 수행
  - 게임플레이: 액션·테크트리·루트·생존·스킬 기반 선택
  - 아트 디렉션: 귀엽고 위트 있는 스타일라이즈드, 코미디 톤
  - 프로덕션: 모바일+데스크톱, 고예산, 12개월 일정 → 팀 구성·간트차트 자동 생성
  - 마케팅: 대중적·넓은 타깃·안전한 캠페인·경쾌한 톤
- **`A2A` 프로토콜**이 에이전트를 분리·집중시키면서도 서로 **통신·협업**하도록 연결
- 중앙 **오케스트레이션 에이전트**가 개별 산출물을 통합해 퍼블리셔 검토용 **제안서**로 조립
- 통합 과정에서 발생한 **게임플레이 ↔ 마케팅 충돌**을 감지하고 사람에게 결정을 요청
- 최종적으로 **아트 스틸, 마케팅 캠페인, 게임플레이 데모 영상**을 자동 렌더링해 다운로드 가능한 결과물로 제공

---

## 💡 개발자 포인트
- **`ADK`로 단일 책임(single-purpose) 에이전트를 설계**하고, **`A2A`로 에이전트 간 협업/통신을 표준화**하는 패턴이 핵심
  - 각 에이전트를 좁고 명확한 역할로 유지하면 디버깅·재사용·확장이 쉬워짐
- **오케스트레이션 에이전트** 패턴: 분산된 산출물을 하나의 결과로 합치는 중앙 조정자가 필요

> ⚠️ 에이전트들이 자율적으로 결정을 내리다 보면 **서로 충돌**하는 선택이 자주 발생한다. 이를 자동 감지해 **human-in-the-loop**로 최종 결정을 넘기는 설계가 실무 agentic 워크플로우에서 필수적이다.

- 텍스트 기획뿐 아니라 **이미지·영상 생성(generative artwork/video)** 을 파이프라인에 통합 가능
- 관련 SDK·ADK 등 기술 정보는 **devsite**에서 확인 가능
