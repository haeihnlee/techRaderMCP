# Tuning custom Gemma models for high speed computer vision

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=PfVjBrgkqpk
- **요약 일시**: 2026-05-28 09:05:22

---

## 🔑 핵심 요약
- 음식물 쓰레기 스타트업 **Mill**(전 Nest 창업자 Matt Rogers)이 **Gemma** 모델을 직접 튜닝해 쓰레기통에 버려지는 음식을 실시간 식별하는 사례 발표
- 클라우드가 아닌 **`NVIDIA Jetson` + 고프레임 카메라** 조합으로 **온디바이스 추론** 구현 — 비용·확장성 문제를 정면 해결
- 식별된 데이터를 **agentic 워크플로우**로 연결해 식당·마트의 발주·메뉴 의사결정까지 개선하는 것이 최종 목표

---

## 📣 주요 발표 내용
- **Mill**은 음식물을 건조·분말화하는 대형 dehydrator 기기로, 카메라가 **초당 120~240 프레임**을 캡처해 버려지는 음식을 분석
- 사람이 프레임 단위로 분석 불가능한 속도 → **컴퓨터 비전 + AI**의 전형적 활용 사례
- 핵심은 **off-the-shelf 모델이 아닌, 손으로 튜닝한 커스텀 `Gemma`** 를 기기 내부에서 직접 실행하는 것
- 학습 데이터는 과거 **Google X**가 전 세계 구글 카페테리아에서 수집한 음식물 데이터 + 자체 수집 데이터, 현재 **5TB 이상의 라벨링 데이터** 보유
- 자체 벤치마킹 툴로 여러 모델을 **mass estimation·식별 정확도** 기준으로 비교 (예: 빨간 사과 vs 토마토 구분 오류 같은 outlier 탐지)

---

## 💡 개발자 포인트
- **클라우드 vs 엣지 트레이드오프**가 핵심 메시지 — Nest 시절엔 영상을 24/7 클라우드 스트리밍했지만, 전 인류의 쓰레기량은 클라우드로 감당 불가 → **엣지 추론이 유일한 현실적 선택**
- 모델 선택은 **태스크 성격에 따라 결정**하라는 실무 가이드:

> 의료처럼 오판이 치명적인 영역은 가장 정교한 deep-thinking 모델이 정당하지만, "사과 vs 토마토"처럼 즉각적이고 오류 정정이 쉬운 rapid-fire 태스크는 가벼운 온디바이스 모델이 합리적이다.

- 단, 작은 오분류도 **누적되면 마트의 발주(procurement) 의사결정을 망칠 수 있음** — "오류 허용"이 곧 "정확도 불필요"는 아님에 주의
- 최종 가치는 모델 그 자체가 아니라, 인사이트를 **agent에 연결해 메뉴 조정·발주 변경·셰프의 문서작업 자동화**로 이어지는 데 있음
