## 🔑 핵심 요약
- **장시간 실행(long-running) 에이전트**가 "한 번의 프롬프트"가 아닌, 며칠에 걸친 **자가 수정(self-correct) 작업 체인**으로 비-trivial 결과물을 만들어내는 시연
- `goal` 프리미티브로 **목표가 충족될 때까지 턴을 반복**시켜, 브라우저용 미니 운영체제(adios)와 3D 비디오 가게 씬을 자동 구축
- 핵심 가치: 에이전트가 중간에 실패해도 **포기하거나 hallucination으로 가짜 fix를 내지 않고** 계속 진행해 완성

## 📣 주요 발표 내용
- **Claude Code**에 단일 spec + 한 줄 지시 → 브라우저에서 도는 운영체제 **adios** 자동 생성
  - 윈도우 매니저, **IndexedDB 파일 시스템**, 터미널, 파일 탐색기 포함
  - 텍스트 에디터(문자 수 카운트), 파일 브라우저, 페인트 앱, 음악 비주얼라이저
  - 운영체제 위에서 **DOOM** 자체 버전까지 구동
- 두 번째 데모: **3D 비디오 대여점 씬**을 비전문가가 에이전트로 구축
  - 출발점은 웹에서 받은 **156MB Blender 파일** → 브라우저 로딩 목표는 5~10MB
  - 에이전트가 직접 **Python 스크립트**를 작성해 변환·최적화 파이프라인 구성

## 💡 개발자 포인트
- 코딩은 대표적인 **long-running workload** — 의존성이 있는 문제들의 연쇄이며 각 단계가 예측 못 한 방식으로 실패할 수 있음
- 에이전트가 **건너뛸 수 없는 하드 기술 문제 3가지**를 순차 해결:

| # | 문제 | 해결 방식 |
| --- | --- | --- |
| 1 | Export 파이프라인 | `Draco` 양자화 시 서로 다른 transform의 메쉬에서 geometry 손상 없이 변환 |
| 2 | 최적화 | `glTF` 거대 파일 + 부속 파일을 **lazy-load** 가능하게 압축, 텍스처 리사이즈·이미지 압축/변환 |
| 3 | 라이팅 | Blender area light의 **과도한 intensity**를 브라우저에서 자연스럽게 보정, material/mixed shader 색 보정 |

> **주의:** Blender area light는 기본값으로 export하면 극단적 intensity로 브라우저에서 깨진 룩을 만듦 — 별도 보정 필수.

- 총 작업 시간: 미니 OS는 수 시간, 3D 씬은 **2~3일(하루 이상)** 연속 실행으로 완성
