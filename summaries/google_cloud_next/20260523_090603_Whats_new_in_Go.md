## 📣 주요 발표 내용

### 생산성 (Productivity)
- **`go fix` 재설계** — Go analysis 프레임워크 기반 modernizer로 20개 이상의 결정론적 코드 변환 제공
  - `go:fix inline` 디렉티브로 deprecated API 호출을 신규 구현으로 일괄 치환
  - 구글 내부 코드베이스에 **18,000+ 커밋** 적용 실적
- **`testing/synctest` 정식 GA (Go 1.25)** — 고루틴을 "bubble" 안에 격리하고 합성 시계로 시간을 자동 진행
  - 5초 sleep 기반 테스트가 **밀리초 단위로 결정론적**으로 종료
- **`new` 빌트인 확장 (Go 1.26)** — 표현식·함수 반환값으로 포인터 생성 가능 (`proto.String` 같은 헬퍼 대체)

### 성능 (Production)
- **Green Tea GC** — 객체 단위 → **페이지(연속 메모리 블록) 단위**로 GC 작업 전환
  - 대부분 앱 GC CPU **10% 절감**, 복잡한 메모리 레이아웃은 최대 **50% 절감**
  - SIMD 활용, NUMA 인식 등 차세대 최적화 기반
- **힙 → 스택 이동** 런타임 최적화 (1.25/1.26)
- **Cgo 호출 30% 빨라짐** (1.26) — ML·게임·GUI 등 신규 워크로드 가능
- **SIMD 1급 지원 (1.26)** — 벡터화 루프 병렬 처리, AI 인프라용 정렬 가속

### AI / 관측 / 보안
- **공식 MCP SDK** 출시 — Go 서비스를 LLM에 도구·컨텍스트로 노출
- **`gopls` MCP 서버 프로토타입** — Go 툴체인을 에이전트에게 개방
- **Flight Recorder** — 링 버퍼 기반 always-on 트레이싱, 필요 시점에만 flush
- **Post-quantum 암호 / 랜덤 힙 주소 / FIPS 140** 강화

## 💡 개발자 포인트

> **Compatibility Promise**: Go 사양에 맞춰 작성된 코드는 모든 향후 릴리스에서 그대로 컴파일·동작. **재컴파일만으로 GC·런타임 개선을 그대로 흡수**할 수 있다.

- **Green Tea GC는 1.26부터 기본값** — 별도 옵션·코드 변경 없이 업그레이드만 해도 GC 비용이 떨어진다. 메모리 집약 서비스는 즉시 검증해볼 가치 있음
- **`go fix` 적극 활용** — 자체 코드베이스 deprecated API 정리에도 `go:fix inline` 디렉티브 추가 시 자동 마이그레이션 가능. AI가 생성한 구식 패턴 코드를 최신 idiom으로 표준화하는 용도로도 적합
- **동시성 테스트는 `synctest`로 재작성** 권장 — `time.Sleep`·타임아웃 기반 flaky 테스트 제거 효과
- **Cgo 비용이 낮아져** 그동안 회피해 온 native lib 연동(ML 추론, GPU, OS API)을 다시 검토할 시점
- **SIMD는 표준 라이브러리 차원에서 지원** — 직접 어셈블리 작성하던 hotpath를 Go 코드로 대체 가능
