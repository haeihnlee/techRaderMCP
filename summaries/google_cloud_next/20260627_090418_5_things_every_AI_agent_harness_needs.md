## 🔑 핵심 요약
- **모델(LLM)은 두뇌일 뿐**이고, 예측을 실제 완료된 작업으로 바꾸는 것은 **하네스(harness)**다.
- 제대로 된 AI 에이전트를 만들려면 하네스에 **5가지 핵심 요소**가 반드시 필요하다.
- 5가지: **컨텍스트 관리 / 툴 라우팅 / 서브 에이전트 / 메모리 / 검증 루프** — 이것이 데모와 실서비스 에이전트를 가른다.

---

## 📣 주요 발표 내용

- **1. 컨텍스트 관리 (Context Management)**
  - 하네스가 매 스텝마다 모델에게 보여줄 내용을 결정
  - 필요한 파일을 로드하고, 너무 긴 내용은 요약하며, 불필요한 것은 버림

- **2. 툴 라우팅 (Tool Routing)**
  - 모델이 `read this file` 같은 행동을 요청하면 하네스가 실제로 수행
  - 툴을 노출하고, 파싱·호출·실행한 뒤 **결과를 다시 모델에 전달**

- **3. 서브 에이전트 (Sub Agents)**
  - 큰 작업은 단일 컨텍스트 윈도우에 다 안 들어감
  - 작업을 분할 → 새 에이전트를 **병렬로 spawn** → 결과를 다시 병합

- **4. 메모리 (Memory)**
  - 단기 기억은 세션 종료와 함께 사라짐
  - 장기 기억은 **디스크에 `skills`·`rules` 형태로 저장** → 없으면 매번 처음부터 시작

- **5. 검증 루프 (Verification Loops)**
  - 모델이 코드를 작성하면 하네스가 실행
  - 출력을 검사하고 **에러를 모델에 피드백**해 스스로 고치게 함

---

## 💡 개발자 포인트

- 에이전트의 성능은 모델 단독이 아니라 **하네스 설계 품질**에 크게 좌우된다.

> **핵심 메시지:** 위 5가지를 제대로 갖춰야 "데모"가 아닌 **실제로 출시 가능한(ships) 에이전트**가 된다.

- 실무 적용 체크리스트
  - 컨텍스트를 무한정 쌓지 말고 **요약·취사선택** 전략을 둘 것
  - 툴 실행 결과를 반드시 모델에 **되먹임(feedback loop)** 할 것
  - 장기 기억을 **디스크(skills/rules)** 에 영속화해 콜드 스타트 방지
  - 코드 생성 후 **자동 실행 + 에러 재투입** 루프를 필수로 구성
