# Give your app search superpowers: Agent Retrieval (Vector Search 2.0)

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=-L5khDHrQSE
- **요약 일시**: 2026-07-17 09:03:34

---

## 🔑 핵심 요약
- Google이 **Vector Search 2.0**을 출시해 임베딩 생성, 인덱싱, 하이브리드 검색을 자동화
- 기존 키워드 검색 대비 **시맨틱 검색(semantic search)** 적용 시 검색 결과가 대폭 증가 (예시: 90만 개 상품 중 키워드 검색 1건 → 시맨틱 검색 30건 이상)
- 인프라·ML 전문 지식 없이도 코드 3단계만으로 구글 규모의 고품질 검색을 구현 가능

---

## 📣 주요 발표 내용
- **Collection** 생성: 데이터 스키마를 정의하고 어떤 필드를 임베딩으로 변환할지 지정 (DB 테이블과 유사한 개념)
- 레코드 로드: 상품 데이터 등을 로드할 때 `vectors` 필드는 비워두면 Vector Search가 자동으로 채움
  - 데이터 소스로 **BigQuery** 또는 Cloud Storage의 텍스트 파일 사용 가능
- 검색 실행 3가지 방식 지원
  - 키워드 검색 (텍스트 매칭)
  - 시맨틱 검색 (임베딩 기반 의도 파악)
  - **하이브리드 검색**: 두 검색을 병렬로 실행 후 `combined` 파라미터로 결과를 하나의 랭킹 리스트로 병합
- SQL과 유사한 **커스텀 필터** 지원 (가격 범위, 카테고리 등 비즈니스 로직 기반 필터링 가능)

---

## 💡 개발자 포인트
- 하이브리드 검색이 대부분의 사용 사례에서 최적의 결과를 제공
- 임베딩 생성·저장·인덱싱 구축을 직접 구현할 필요 없이 Vector Search 2.0이 처리
- 대규모 데이터셋과 트래픽에도 자동으로 스케일링됨

> 별도의 인프라 전문가나 머신러닝 엔지니어 없이도 프로덕션급 시맨틱 검색을 구축할 수 있다는 점이 핵심 메시지

- 관련 블로그 포스트와 실습용 노트북이 공개되어 있어 직접 코드를 실행하며 학습 가능 (영상 설명란 참고)
