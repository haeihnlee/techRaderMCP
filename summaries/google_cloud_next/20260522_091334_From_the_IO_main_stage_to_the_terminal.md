## 📣 주요 발표 내용
- **Emergent 플랫폼 아키텍처**
  - 모바일·웹 풀스택 앱을 프롬프트만으로 생성 후 **end user에게 직접 배포**
  - 디자인 → E2E 테스트 → 백엔드 → DB까지 엔지니어 워크플로우를 그대로 내재화
  - 서드파티 샌드박스 미사용, **GKE 기반 자체 컨테이너**로 에이전트가 DB·서버 상태를 로컬에서 실시간 감지
- **모델 오케스트레이션 전략**
  - 작업 유형별로 모델 자동 분기: 프론트엔드 UI는 **Gemini**, 무거운 추론은 **Gemini 3 Pro**, 빠른 응답·툴콜은 **Gemini Flash**
  - `Nano Flash`, `Veo 3` 등 AI 기능을 사용자가 **API 키 없이** 프롬프트로 통합 가능
- **신제품 'Wingman' (코드네임)**
  - 모바일에서 **텍스트만 보내면** 앱이 빌드되는 on-the-go 빌더

## 💡 개발자 포인트
- **수동 eval을 과소평가하지 말 것** — 첫날부터 자동화에 매달리지 말고, 휴먼 게이트키퍼로 'vibe sense'부터 확보한 뒤 자동 eval로 확장
- **온라인 라이브 테스트**: 프로덕션에서 신모델이 전환율·리텐션 같은 비즈니스 지표를 실제로 개선하는지 통계적 유의성으로 검증
- Google의 **experiment velocity** 문화를 스타트업에 이식 — 모델 교체 시 의사결정의 신뢰도 향상

> ⚠️ **반직관 포인트**: 더 스마트한 모델이 **토큰 단가가 비싸 보여도 실제 비용이 낮을 수 있다.** 똑똑한 모델은 삽질을 덜 해서 총 토큰 소비량이 줄어듦 → **토큰당 가격이 아닌 '작업당 비용'으로 평가**할 것

- **에이전트 인프라 = 코딩 에이전트 + 실행 환경**: 코딩 에이전트만으로는 부족하고, 에이전트가 자기 출력의 결과(빌드 실패·DB 다운)를 로컬에서 즉시 받을 수 있는 샌드박스가 핵심
- 비개발자(도메인 전문가)가 주말에 Airbnb 클론을 만든 사례 — **$100K 외주 견적이 셀프서비스로 대체**되는 시장 전환
