## 🔑 핵심 요약
- **BigQuery Graph**는 기존 BigQuery 데이터셋에 그래프 분석 기능을 직접 통합한 새로운 기능
- 데이터 복사 없이 **기존 테이블**을 그대로 활용하여 그래프 구조를 생성 가능
- **GQL(Graph Query Language)** 을 통해 관계형 데이터와 그래프 분석을 동시에 활용

---

## 📣 주요 발표 내용

- 🆕 **BigQuery Graph** 도입: 관계형 DB의 구조적 장점 + 그래프 DB의 연결 관계 분석을 하나의 플랫폼에서 제공
- 🗂️ 기존 테이블(예: `accounts`, `customers`, `transactions`)을 **노드(Node)** 와 **엣지(Edge)** 로 변환하는 단일 명령어 지원
- 🔍 **GQL(Graph Query Language)** 을 사용한 그래프 쿼리 지원
- 🤝 **SQL과의 상호 운용성(Interoperability)** 지원 — 기존 SQL 워크플로우와 병행 사용 가능
- 🧠 **벡터 검색(Vector Search)** 및 **전문 검색(Full-Text Search)** 등 고급 기능과 통합 가능
- 🤖 **AI 에이전트** 강화에도 활용 가능한 그래프 표현 구조 제공

---

## 💡 개발자 포인트

- **데이터 복제 불필요**: 기존 BigQuery 테이블을 그대로 사용하여 그래프 뷰를 생성
  - 스토리지 비용 추가 없이 그래프 분석 도입 가능

- **주요 활용 시나리오**:
  - 💳 금융 사기 탐지 (fraudulent transaction detection)
  - 📦 공급망(Supply Chain) 장애 영향 범위(blast radius) 분석
  - 🧑‍🤝‍🧑 고객/직원 소셜 그래프 모델링
  - 🏭 부품 → 완제품 영향도 분석

- **GQL 명령어** 한 줄로 그래프 생성:
  ```sql
  -- 예시 구조 (개념적 표현)
  CREATE PROPERTY GRAPH my_graph
    NODE TABLES (customers, accounts)
    EDGE TABLES (transactions);
  ```

> ⚠️ **주의**: BigQuery Graph는 기존 SQL 쿼리를 대체하는 것이 아니라 **보완하는 기능**입니다. SQL과 GQL을 함께 사용하는 혼합 쿼리 패턴을 이해하고 설계해야 합니다.
