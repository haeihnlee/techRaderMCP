# Building an AI app: A low-code guide for small teams

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=hOQ_pIFSvh0
- **요약 일시**: 2026-06-26 09:10:36

---

## 🔑 핵심 요약
- 소규모 팀이 **기존 앱에 AI를 빠르게 추가**하는 로우코드 전략을 소개 — 핵심은 "데이터가 있는 곳에 AI를 얹는 것"
- 별도의 전용 벡터 DB·ETL 없이 **모던 DB에 내장된 AI 기능**(`AlloyDB` 등)만으로 시맨틱 검색 구현 가능
- 시작을 위한 4가지 권장 요소: **바이브 코딩**, **하이브리드 검색**, **MCP 서버/도구**, **챗봇**
- 데모: 플랜트 쇼핑몰의 전문(full-text) 검색을 **프롬프트 한 번으로 시맨틱 검색으로 전환**

---

## 📣 주요 발표 내용
- **소규모 팀의 3대 요구사항**: 쉬움(easy), 저렴함(cheap), 보안(secure) — 전담 보안 인력 없이도 베스트 프랙티스를 따라갈 수 있어야 함
- **DB가 AI 앱의 핵심**: 최근 2~3년간 출시된 DB는 AI 모델(`Gemini` 등)과 앱/에이전트 사이의 간극을 메우는 기능을 내장
- **AI 활용 2가지 관점**
  - `build time` — 개발자 보조(앱을 빠르게 만들거나 확장). 최종 사용자 경험에는 AI가 드러나지 않음
  - `run time` — 최종 사용자가 직접 경험하는 에이전트/AI 경험 구축
- **엔터프라이즈 바이브 코딩 3요소**
  - **Agent Skills** — 로컬 컨텍스트를 주입해 에이전트가 특정 작업을 수행하도록
  - **MCP 서버/도구** — 에이전트의 데이터·앱 접근 범위와 방식을 통제하며 권한 부여
  - **Plugins & Extensions** — VS Code 코드를 에이전트 경험으로 전환, 토큰 사용 최적화
- **데모 (시맨틱 검색 전환)**: `Gemini` + `AlloyDB`(매니지드 Postgres)를 **매니지드 원격 MCP 서버 엔드포인트**로 연결
  - "pet friendly" = "dogs/cats friendly"를 이해하지 못하던 전문 검색의 한계를 시맨틱 검색으로 해결

---

## 💡 개발자 포인트
- 프롬프트만으로 에이전트가 ① 기존 테이블 스키마 검사 → ② 임베딩용 컬럼 추가 → ③ 인덱스 생성 → ④ 프론트엔드 업데이트까지 자동 수행
- 데모에서 에이전트가 실제로 수행한 작업
  - `description_vector` 벡터 컬럼 생성
  - `text-embedding-005` 모델로 임베딩 생성 (AlloyDB AI 스키마 활용)
  - 해당 컬럼에 **고성능 ScaNN 인덱스** 추가
  - API·백엔드·프론트엔드를 시맨틱 검색용으로 갱신

> **실전 팁**: 기존 검색을 곧바로 교체하지 말고, 프롬프트에서 "토글로 A/B 비교 가능하게 둘 다 유지"를 명시하면 안전하게 전환·검증 가능

> AI 앱을 위해 반드시 특수 벡터 DB와 대규모 ETL이 필요하다는 것은 **흔한 오해(myth)** — 데이터가 이미 있는 곳에 AI 기능을 얹는 것이 가장 빠른 길
