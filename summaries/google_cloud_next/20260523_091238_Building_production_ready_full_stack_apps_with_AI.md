## 🔑 핵심 요약
- **Emergent**는 비개발자도 풀스택 앱을 프로덕션 레벨로 배포할 수 있게 해주는 **AI App Builder**로, 공개 출시 1년 만에 **ARR $100M**, **앱 1천만 개 배포**, **사용자 900만 명** 달성
- 자체 **Sandbox Container 기술**을 **GKE(Google Kubernetes Engine)** 위에 직접 구축, 에이전트가 DB/백엔드 상태를 실시간 피드백받아 사람 개입 없이 자가 수정
- **Gemini Flash**(프론트 UI·tool calling) ↔ **Gemini 3 Pro**(heavy task)를 작업 단위로 자동 라우팅하여 비용·성능 최적화

---

## 📣 주요 발표 내용
- **Emergent 코딩 에이전트**: SWE-bench Verified 벤치마크에서 1위를 기록한 바 있는 자체 에이전트로, 모바일/웹 풀스택 앱을 코드·디자인·테스트·DB·백엔드까지 일괄 생성
- **자체 인프라 스택**
  - 서드파티 sandbox 미사용, **GKE 기반 자체 컨테이너 런타임** 구축
  - 에이전트 실행 환경(DB·백엔드)이 **로컬 머신에서 실시간 시그널 수집** → 사람 개입 없이 자율 디버깅
- **모델 오케스트레이션**
  - 작업 유형별 모델 자동 선택 (`프론트 UI` → `Gemini Flash`, 무거운 작업 → `Gemini 3 Pro` 등)
  - `Nano Banana`(Nano Flash), `Veo 3` 등 멀티모달 모델을 **API Key 발급 없이** 프롬프트만으로 통합 가능
- **Wingman (코드네임)**: 모바일에서 **문자(text)로 앱을 빌드**하는 신규 제품. Flash의 속도 덕에 이동 중 아이디어를 즉시 앱으로 변환
- **평가(Eval) 체계**: 수동 평가 → 자동 평가 → **온라인 라이브 A/B 테스트** 3단계 파이프라인으로 새 모델을 빠르게 검증

## 💡 개발자 포인트
- **모델 가격은 토큰 단가가 아니라 "작업당 비용"으로 봐야 한다**
  - 더 똑똑한 모델은 시도 횟수가 줄어 결과적으로 **저렴**해질 수 있음 → 본인 워크로드로 직접 측정 필수

> ⚠️ **자동화 함정 주의**: 많은 팀이 Day 1부터 자동 eval만 도입하려 하지만, **수동 eval(human-in-the-loop)을 먼저 돌려 "vibe"를 확보**한 뒤 자동화로 옮겨가는 것이 모델 변경 대응에 훨씬 강건함

- **풀스택 AI 빌더 설계 인사이트**
  - 코딩 에이전트 + **샌드박스 런타임**을 함께 소유해야 진정한 자율 디버깅이 가능 (서드파티 sandbox로는 실시간 상태 시그널 제한)
  - 단일 모델 락인 X → **태스크별 모델 라우팅 레이어**를 직접 구축할 것
- **타겟 유저 재정의**: "Idea Guy"가 AI 시대에 가장 강력한 사용자로 부상. 도메인 전문가(부동산·은퇴자 등)가 주말 만에 Airbnb 클론을 만드는 사례 다수 → 비개발자 친화 UX가 핵심 차별점
