## 🔑 핵심 요약
- **Antigravity 2.0**(에이전트형 AI 코딩 툴)을 신규 데모가 아닌 **대규모 엔터프라이즈 코드베이스**에 적용하는 실전 7가지 전술 정리
- 핵심 마인드셋 전환: 단순 "프롬프터"가 아니라 에이전트를 **코칭하는 코치**로 일할 것 (영상에선 에이전트를 인턴 "Ducky"에 비유)
- **자가 수정 루프(self-correction loop)**, **백그라운드 비동기 테스트**, **브라우저 검증**, **Mermaid 다이어그램**, **PR 사전 리뷰** 등으로 개발 생산성 극대화

## 📣 주요 발표 내용

- **Tip 1 — 팀의 DNA를 커스텀 Skill에 인코딩**
  - 클린 코드·디자인 패턴·팀 표준을 매번 반복 지시하지 말고 **커스텀 Skill**로 작성
  - TDD를 쓰면 전용 **TDD Skill**을 만들 것. Skill은 단순 **마크다운 파일**이라 직접 작성이 쉬움
  - 인터넷에서 받은 랜덤 Skill은 쓰지 말 것

- **Tip 2 — Paper trail(문서 흔적) 남기기**
  - 코드 수정 전 **구현 계획서(implementation plan)** 자동 생성 활용
  - 거기서 멈추지 말고 **테스트 계획·아키텍처 스펙** 문서도 생성 → 코드 작성 전 댓글로 방향 조정
  - 생성한 계획서를 **브랜치에 저장**하면 에이전트가 읽고 더 똑똑해지고, 리뷰어에게도 도움. PR 머지 후 삭제

- **Tip 3 — 자가 수정 루프 (가장 큰 시간 절약)**
  - 사람을 피드백 루프에서 **제거**해 가장 타이트한 루프 구성
  - 린터·타입 체커·유닛/통합 테스트·골든 스크린샷 명령을 **프로젝트 룰 / README / Skill**에 등록
  - 에이전트가 테스트 실행 → 에러 읽기 → 코드 수정 → 통과까지 **스스로 반복**

- **Tip 4 — 에이전트에게 "눈"을 주기**
  - 프론트엔드/복잡한 UI는 텍스트 로그만으론 부족 → **내장 브라우저 툴** 사용
  - 에이전트가 직접 앱을 열어 깨진 레이아웃·프론트 버그를 사전 수정
  - 브라우저가 안 열리면 프롬프트에 `/browser` 추가, 또는 **스크린샷 붙여넣기**로 대체

- **Tip 5 — 느린 작업은 비동기(async)로**
  - 무거운 통합 테스트·E2E 브라우저 테스트는 **백그라운드 async task**로 실행
  - 에이전트가 멈추지 않아 워크스페이스를 비워두고 다음 작업 진행 가능

- **Tip 6 — Mermaid 다이어그램으로 아키텍처 시각화**
  - Antigravity 2.0이 마크다운 기반 **Mermaid 다이어그램**을 툴 내에서 생성·표시
  - ① 모듈 호출 관계 다이어그램으로 **순환 의존성·잘못된 레이어 호출** 적발
  - ② 신규 기능 설명 다이어그램을 **GitHub PR 설명**에 붙여 리뷰어에게 시각적 개요 제공

- **Tip 7 — 코드 리뷰어 부담 줄이기 (사전 리뷰 + 리팩토링)**
  - PR 제출 전에 **사전 리뷰·리팩토링**으로 리뷰어 부하 경감
  - 재사용 Skill 구성: 보안 취약점 검사 / 올바른 idiom 검사 / 코드베이스 라이브러리 일관성 검사
  - 컴플라이언스 리포트를 받아 **클린해질 때까지 가차없이 리팩토링**. 개발 시간의 50% 이상을 리팩토링에 투자

## 💡 개발자 포인트

- AI 시대에 **코드 작성은 쉬워졌지만 리뷰는 더 어려워졌다** → 제출 전 사전 정제가 핵심 전략
- 에이전트가 스스로 채점하려면 **검증 명령(linter/typecheck/test)을 프로젝트 룰·Skill에 명시**하는 것이 전제 조건

> 많은 사내(corporate) 환경은 보안이 잠겨 있어 **브라우저 툴이 실행되지 않을 수 있음**. 이 경우 지시문과 함께 **스크린샷을 붙여넣어** 대체하라.

> Skill·룰·계획서 대부분이 그냥 **텍스트 파일**이라 진입 장벽이 낮음 — "오늘 바로 시작 가능"하다는 점이 핵심 메시지.
