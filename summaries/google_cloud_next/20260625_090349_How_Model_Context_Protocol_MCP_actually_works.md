# How Model Context Protocol (MCP) actually works

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=cGuyrANVi4A
- **요약 일시**: 2026-06-25 09:03:49

---

## 🔑 핵심 요약
- **MCP(Model Context Protocol)**는 모델과 도구·데이터·컨텍스트를 **일관된 방식으로 연결**하는 오픈 표준이다.
- 매번 통합 코드를 하드코딩하던 방식 대신, 모델이 사용 가능한 도구를 **동적으로 탐색(discover)**하고 호출할 수 있게 한다.
- `HTTP`가 웹을 통일했듯, **MCP가 모델-도구 연동 방식을 표준화**하고 있어 지금 익혀둘 가치가 크다.

---

## 📣 주요 발표 내용
- **MCP는 Anthropic이 제안**한 오픈 표준으로, 현재 업계 전반으로 채택 확산 중이다.
- 프로토콜은 **`client`(모델/에이전트)** 와 **`server`(리소스 제공 환경)** 두 축으로 구성된다.
- 클라이언트 예시: **Claude**, **Gemini app**, OpenAI·Google 프레임워크 기반 에이전트.
- 서버 예시: 데이터베이스, 파일시스템, 사내 도구, 문서 검색 엔진 등.
- 서버는 단순 데이터 응답이 아니라 **자신의 capability(지원 동작·리소스·필요 입력)를 광고(advertise)**한다.
- 통신은 단순하고 명확한 스키마로 이뤄짐 — 클라이언트가 `list available resources`, `call this action`, `retrieve this data` 등을 요청하면 서버가 구조화된 **JSON**으로 응답한다.
- MCP가 정의하는 핵심 리소스 타입 4가지:
  - **`Tools`**: 모델이 호출하는 동작 (DB 검색, 이메일 발송, 파일 분석 등)
  - **`Resources`**: 데이터·상태 (문서, DB 레코드, 이미지 등)
  - **`Prompts`**: 특정 작업용 재사용 템플릿
  - **`Context`**: 추론에 끌어오는 외부 정보 (대화 이력, 사내 데이터, 사용자 선호)
- 각 도구·리소스에는 **무엇을 하는지·입력·출력**을 설명하는 메타데이터가 포함된다.

## 💡 개발자 포인트
- **MCP는 API의 상위 추상화 레이어**다 — 기존 `REST`/`GraphQL` API는 그대로 두고, MCP 서버가 내부에서 호출한다. 모델은 통일된 MCP 스키마로만 상호작용한다.
- API는 *이미 무엇을 원하는지 아는 프로그램*을 위해 설계됐지만, **LLM은 확률적으로 추론**하므로 탐색·질의·검증이 가능한 인터페이스가 필요하다 — 이 간극을 MCP가 메운다.

> 핵심 이점: 도구마다 **glue 코드를 새로 작성할 필요가 없다.** 프로토콜이 일관되므로, 한 번 MCP 서버로 노출하면 **모든 호환 모델이 커스텀 통합 없이** 사용할 수 있다.

- 실전 예시 — 일정 확인·회의 노트 추출·후속 이메일 초안 작성 에이전트:
  - **기존 방식**: Google Calendar·Notion·Gmail API를 각각 통합, 인증·rate limit·엣지케이스 처리 + 길고 깨지기 쉬운 시스템 프롬프트로 사용법 학습.
  - **MCP 방식**: 시스템별 서버(calendar/notion/email)를 설치하면 각 서버가 `list events`·`get meeting summary`·`send email` 같은 도구를 자동 광고 → 모델이 **무엇을·어떤 순서로·어떤 데이터로** 쓸지 스스로 추론.
- `GitHub`·`CRM`·`calendar` 등 어떤 서버를 쓰든 **동일한 스키마(언어)**로 말하므로 일관성이 보장된다.
