# Automate SAP order-to-cash with multi-agent AI & Google Agent Development Kit (ADK)

- **컨퍼런스**: Google Cloud Next
- **출처**: https://www.youtube.com/watch?v=z4CXn4CKHQI
- **요약 일시**: 2026-06-03 09:05:21

---

## 🔑 핵심 요약
- **Google ADK**(Agent Development Kit)와 **MCP**(Model Context Protocol)로 **SAP**의 반복적인 주문 처리(`order-to-cash`) 업무를 멀티 에이전트로 자동화하는 사례
- 기존 **RPA**(화면 클릭·입력 모방) 방식의 한계를 벗어나, SAP의 **OData API**를 직접 호출하는 안정적인 자동화로 전환
- **루트 에이전트(supervisor)** 아래 **하위 에이전트**(재고 담당·주문 생성 담당)를 두어 회사 조직의 수직·수평 위계를 그대로 에이전트 팀으로 매핑

---

## 📣 주요 발표 내용
- **문제 정의**: 정유·자동차·헬스케어 등 대기업은 하루 수천 건의 주문을 처리 → 자동화 시 운영비 절감·효율 향상
- **레거시 한계**: **RPA**는 화면 해상도·UI 변경에 쉽게 깨져 개발자가 계속 보수해야 함
- **새로운 접근**: SAP가 **`OData`** 표준으로 노출하는 API/엔드포인트 → **MCP 도구**로 래핑 → **ADK 에이전트**가 도구로 활용·오케스트레이션
- **도구 자동 생성**: **`OpenAPI MCP Generator CLI`** 로 OpenAPI 스펙에서 MCP 서버·도구를 자동 생성 (sales order API 예시)
- **에이전트 팀 구성**
  - **Sales Supervisor** (루트/코디네이터): MCP 도구에 직접 접근하지 않고 하위 에이전트를 지휘
  - **Inventory Specialist** (재고 조회)
  - **Sales Order Creator** + **Reviewer** (주문 생성·검토): 각자 전용 프롬프트와 도구 셋 보유
- **실제 데모**
  - 주문번호 `6319` 조회 → Reviewer 에이전트가 `get sales order` MCP 도구 호출, 필요한 필드만 **필터링**해 reasoning 수행
  - 미국 고객용 신규 주문 생성 → Creator 에이전트가 여러 MCP 도구를 연쇄 호출, 신규 주문번호 `6321` 생성
  - 총액 질의 → 팀이 SAP 조회 후 **$208,000** 산출

## 💡 개발자 포인트
- **OpenAPI → MCP 자동 변환**이 핵심: SAP의 SD(영업/유통)·MM(자재관리) 모듈만 해도 수백 개 API가 있어 MCP 도구도 수백 개로 폭증 → 수작업 대신 CLI 생성기로 스케일 대응
- **계층형 에이전트 설계**: ADK 에이전트는 하위 에이전트에 작업을 위임하거나, 다른 에이전트를 도구처럼 사용 가능 → 기존 조직 역할·책임을 재설계 없이 그대로 매핑
- **Gemini 1M 토큰 컨텍스트 윈도우**가 수백 개 도구·대규모 스키마를 다루는 데 결정적 역할
- 에이전트는 단순 파라미터 전달이 아니라 **필터 조건을 스스로 추론**해 호출 → 도구 호출의 품질이 LLM reasoning에 좌우됨

> **ADK 1.1부터 에이전트가 도구를 병렬(parallel)로 호출**할 수 있어 성능이 향상됩니다. 데모에서 주문 아이템 2건을 한 번에 처리한 것이 이 기능 덕분입니다.

> RPA에서 API 기반 자동화로 전환하면 화면 변경에 깨지지 않지만, **SAP 권한·API 스코프 설정**이 선행되어야 실제 쓰기(주문 생성)가 가능합니다.
