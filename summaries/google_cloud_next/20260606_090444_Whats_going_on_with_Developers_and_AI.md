## 🔑 핵심 요약
- 지난 4년간 개발 방식은 **"코드를 한 글자씩 직접 작성"** → **"자연어로 의도를 기술하면 AI가 코드 생성"**으로 근본적으로 전환됨
- 개발자의 핵심 역량이던 **"무엇을 어떻게 쓸지 아는 것"**이 상당 부분 자동화되면서, 흥분과 두려움이 공존하는 시대
- 구글 `aistudio.google.com`에서 브라우저만으로(설치·코딩 경험 불필요) 몇 문장만으로 앱 전체를 생성하는 **바이브 코딩(vibe coding)** 시연

## 📣 주요 발표 내용

**프로그래밍 워크플로우의 시대별 변천사**

- **1980년대**: `TRS-80` + `BASIC` — `10 PRINT "hello world"` / `20 GOTO 10` 처럼 6살도 따라 할 만큼 접근성이 높았던 시절
- **2000년대~2022년**: 코드 작성 → 막히면 **Google 검색** → **Stack Overflow** 코드 복붙 → 내 문제에 맞게 수정하는 표준 워크플로우
- **2022년**: AI에 구체적 질문 → 내 문제에 맞춤화된 코드를 바로 받아 붙여넣으면 동작하는 시대 개막

**AI의 IDE 통합 단계**

- 1단계: **Tab 자동완성** — LLM이 입력 중인 줄/다음 몇 줄을 예측해 `Tab`으로 완성
- 2단계: **`Cursor`** — 채팅을 IDE 안으로 가져와, 전체 파일시스템·코드베이스를 **컨텍스트로 자동 주입** → 더 개인화된 답변
- 3단계(2026, 현재): **`aistudio.google.com`** — 마이크로 말하면 자연어를 정제해 **앱 전체를 1~2분 만에 빌드**

## 💡 개발자 포인트

- AI Studio 시연 흐름: **Build 버튼 → 만들 것 음성/텍스트 입력 → 빌드** 진행 중 좌측 chat에서 사고 과정 확인, `metadata.json`·스타일시트 등 생성 코드를 **Code 탭**에서, 결과는 **Preview 탭**에서 확인 가능
- 별도 앱 설치 없이 **모바일 브라우저에서도** 바이브 코딩 가능

> 발표의 핵심 메시지: 변화가 두렵다면 가장 좋은 해법은 **직접 손을 더럽히며(get your hands dirty) 만들어보는 것**. 재미·업무용 무엇이든 일단 빌드해보면 두려움보다 재미가 크다는 것을 알게 됨.

> 개발자의 정체성 변화: 한 글자씩 깎던 **'장인(craftsman)'**에서, 처음으로 **'전동 공구(power tools)'**를 손에 쥔 시대로. "손톱을 다시 안 써도 되나?", "이제 10배 더 많이 만들어야 하나?" 같은 질문은 아직 미해결 상태.
