## 🔑 핵심 요약
- **AI Studio**에서 프롬프트만으로 웹앱을 생성하고 `Cloud Run`으로 원클릭 배포하는 **바이브 코딩** 워크플로우 소개
- 프로토타입을 프로덕션으로 올리는 "Day 2 문제"의 핵심은 **데이터베이스**와 **인증** — AI Studio에 `Cloud SQL`(PostgreSQL)·`Firestore` **원클릭 통합**이 추가됨
- 데모: 바이브 코딩으로 만든 아이스크림 게임에 사용자 인증 + 하이스코어 저장(리더보드)을 DB 통합으로 구현

---

## 📣 주요 발표 내용
- **AI Studio DB 통합**: 수동 설정 없이 에이전트가 DB 연동을 자동 처리
  - `Cloud SQL for PostgreSQL`(관계형): 에이전트가 **스키마를 자동 생성**하고 SQL 문을 직접 실행
  - `Firestore`(NoSQL 문서형): 에이전트가 **문서 모델과 쿼리를 자동 정의**
- **인증(Authentication)** 추가도 원클릭으로 지원 — 엔터프라이즈 배포의 필수 요소
- 관계형 vs 비관계형 선택 가이드: 구조가 명확한 데이터(유저-포스트-댓글)는 관계형, 스키마가 유동적인 앱은 문서형 — AI Studio 에이전트에게 물어보면 앱 특성에 맞는 **추천**도 받을 수 있음
- **Firestore 장점**: 스키마 변경 시 다운타임 없음, 높은 확장성·가용성(Black Friday급 트래픽 대응), **실시간 동기화**(폴링·새로고침 없이 UI 자동 갱신), `MongoDB` 호환 기능 제공

---

## 💡 개발자 포인트
- 바이브 코딩으로 만든 프로토타입도 결국 **DB·인증·쿼리 설계**를 피할 수 없음 — AI Studio 통합이 이 진입장벽을 크게 낮춤
- `Firestore`는 초보자 친화적: 스키마 사전 설계 부담이 적고, 리더보드 같은 단순 저장부터 시작해 그대로 스케일업 가능
- 자주 실행할 쿼리가 있다면 `Firestore` **인덱스 설계**는 여전히 직접 고려해야 함
- 실시간 동기화는 리더보드·협업 UI에 유용하지만, 위치 공유처럼 고빈도 양방향 통신은 **WebSocket** 계열이 더 적합할 수 있음
