## 📣 주요 발표 내용

### 1. 기획 단계
- **MVP(Minimum Viable Product)** 중심 사고: 카카오톡·당근마켓도 초기엔 단 하나의 핵심 기능만 있었음
- **타겟 페르소나**를 구체적·명확하게 정의 → 서비스 성공률 결정
- **경쟁 서비스 분석**으로 차별화 포인트 도출
- 기획은 거창한 게 아니라 **"무엇을 만들지 생각하는 것"** 자체부터 시작

### 2. 설계 단계
- **화면 설계 (UI/UX)**: 사용자 시나리오 + 화면 상세 설계
- **데이터 설계 (ERD)**: 회원·상품·결제 등 테이블과 관계 정의
- **API 설계**: 앱/웹은 DB에 직접 접속 못 함 → API 서버가 중간자 역할

### 3. 개발 단계
| 영역 | 기술 | 핵심 작업 |
|------|------|----------|
| **프론트엔드** | `HTML`/`CSS`/`JS`, iOS/Android | 화면, 이벤트 처리, 서버 통신 |
| **백엔드** | API 서버 | `CRUD`(Create/Read/Update/Delete), 로직, 예외 처리 |
| **DB** | `MySQL`, `Oracle`, `MongoDB` | DBMS 설치, 테이블 생성, SQL |

- **프론트-백엔드 연동**: API 호출/응답 검증
- **회원가입/소셜 로그인** 구현
- **반응형 웹** 또는 네이티브 앱(iOS/Android) 분기

### 4. 테스트
- 단위 / 통합 / 성능 / 보안 / 설치 테스트

### 5. 소스 관리 & 빌드
- **`git`은 필수** — 취업 시 반드시 숙지
- **브랜치 전략**: 개발 / 운영 분리 관리

### 6. 배포(Deploy)
- **클라우드** (`AWS`, `Azure`, `GCP`) — 사용자/서버 확장 대비 권장
- **간편 배포 도구**: `Vercel`, `Cloudflare` 등 (단순 웹 서비스에 적합)

### 7. 출시 준비
- **도메인 구매 → 서버 IP와 연결**
- **SSL 인증서 설치 → HTTPS 설정 (필수)**
- **앱스토어 / Google Play 등록 + 심사 (수 일 소요)**

### 8. 운영(유지보수)
- 장애 대응 / 복구
- 기능 추가 / 개선
- 코드 베이스 지속 관리

## 💡 개발자 포인트

> 💡 **"출시는 끝이 아니라 시작이다"** — 운영 단계가 가장 길고 까다로움. 처음부터 운영을 염두에 둔 설계가 중요.

> ⚠️ **소스/서버는 반드시 분리하라**: 운영 중인 소스를 직접 수정하는 것은 매우 위험. 개발 → 테스트 → 운영 반영의 단계적 프로세스를 지켜야 함.

> 💰 **수익화는 기획 단계부터 고민할 것** — 출시 후 추가하는 게 아니라 처음부터 비즈니스 모델로 설계해야 함.

- **API ↔ DB 테이블의 밀접한 관계**: API는 결국 테이블 데이터를 채우거나 읽어오는 인터페이스 → 두 설계는 동시에 진행
- **욕심 버리기**: 기획 단계에선 기능을 늘리는 게 아니라 **포커스를 좁히는 작업**
- **클라우드 vs 간편 배포 도구 선택 기준**
  - 단순 웹 서비스 → `Vercel`/`Cloudflare`로 충분
  - DB·API 다수 사용 → 클라우드 (`AWS`/`Azure`/`GCP`) 필요
