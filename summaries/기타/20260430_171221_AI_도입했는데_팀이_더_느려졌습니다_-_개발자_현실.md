## 🔑 핵심 요약
- 캐나다 모바일 엔지니어링 매니저의 현장 후기 — **Cursor 도입 후 PR 생산 속도는 올랐지만 배포는 오히려 더 느려진** 역설
- 코드 작성 속도 ↑ → **리뷰·CI 파이프라인 병목** ↑ → 전체 사이클은 더 길어지고 품질은 떨어짐
- 진짜 차별화는 **AI를 코드 생성이 아닌 팀 프로세스·파이프라인 개선에 활용하는 사람**

## 📣 주요 발표 내용

### 도입 후 발생한 병목 체인
- **Cursor**로 PR 생산 속도 폭증 → 리뷰어 부족
- 자체 **AI 코드 리뷰 에이전트** 도입 (PR마다 critical 코멘트 5개 필수 수정 후 머지)
- **CI 파이프라인 부하**: 유닛 테스트 + Security review + Firebase test + TestFlight + 배포 → 변경분만이 아닌 **전체가 매번 실행**, 1~2시간 소요
- 결과 사이클: 코드 수정 → 커밋 → 1~2시간 파이프라인 → 리뷰 → AI 에이전트 새 코멘트 → 다시 수정 (무한 반복)

### 부수적으로 깨진 것들
- **앱 일관성 붕괴**: 여러 도메인에서 동시 빠르게 작업 → UX 불일치 → 프로덕트팀 정리 사이클 추가
- **리뷰 품질 저하**: 사이클 압박에 제대로 안 보고 그냥 머지하는 케이스 증가
- **Cursor의 한계**: 작은 코드 블록은 강함, **아키텍처/구조적 문제는 도움 안 됨** — 팀원 대다수가 이 차이를 모름

### 거시적 배경 (캐나다 기준)
- 고금리 + 펀딩 위축 + AI 툴 비용 상승
- 경영진 기대치 ↑, 예산 ↓ → "AI 있는데 왜 느려?" 압박
- AI를 잘 쓰는 환경을 만드는 책임자가 부재 (누구의 일도 아님)
- 채용 인터뷰는 여전히 **코딩테스트 + 시스템 디자인 + 행동 인터뷰** — AI 활용 능력을 검증하지 않음

### 시장 시그널
- 캐나다에서 **계약직 채용 급증** (3일/1일 노티스로 해고 가능 → 적정 인원 탐색)
- 2026년 2분기 들어 다시 채용 증가 — 경영진이 AI 활용도와 현장의 갭을 인지

## 💡 개발자 포인트

> **"AI가 개발자를 대체하는 게 아니라, AI를 잘 쓰는 개발자가 못 쓰는 개발자를 대체한다."**

**병목 = 기회.** 아무도 안 하는 영역에 임팩트가 있음:
- **Pre-commit hook으로 코드 리뷰 에이전트 로컬 실행** → PR 올리기 전에 품질 향상 → 사이클 단축
- **CI 파이프라인 최적화**: 변경된 모듈만 실행 / 데드 코드 제거 — 하루 이틀이면 가능한 작업
- AI를 **코드 생성용**이 아닌 **팀 프로세스·파이프라인 개선용**으로 쓰는 사람이 가장 차별화됨

> ⚠️ **매니저의 시선 (현실)**
> AI 툴을 깊이 안 쓰거나 거부하는 팀원은 **매니저 레이더에 걸림**. 커뮤니케이션·툴 활용 못 하면 존재감 사라짐.

**살아남는 타입**:
- AI의 한계를 정확히 아는 사람 (뭘 잘하고 뭘 못하는지)
- "AI가 다 해줄 거야" / "AI 쓸모없어" 양극단 모두 아닌 사람
- AI 툴 하나에 의존하지 않고 **여러 도구를 써보며 작동 원리를 이해하고 자기 워크플로에 녹이는 사람**

> **잠재적 변수**: Anthropic 내부에 보안 이유로 미배포된 강력한 모델(영상에서 "미소스"로 언급)이 풀리면 시장이 다시 흔들릴 가능성
