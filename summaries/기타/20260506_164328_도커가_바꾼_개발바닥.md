## 🔑 핵심 요약
- **Docker**는 OS 레벨 가상화 기술로, "코드 밀키트"를 만들어 어디서나 동일하게 실행할 수 있게 해주는 도구
- 기존 **VM** 대비 가볍고 빠른 **리눅스 컨테이너** 기반으로, 개발자가 가장 많이 쓰는 툴 1위
- **MSA**, **자동 배포(CI/CD)**, 로컬 환경 격리 등 현대 개발 워크플로우의 핵심 인프라로 자리잡음

---

## 📣 주요 발표 내용
- **컨테이너 이전 시대의 문제점**
  - 새 직장 입사 시 개발 환경 세팅에 하루 종일 소요
  - "내 컴퓨터에선 잘 되는데" 문제로 배포 시 잦은 장애
  - 환경 업데이트마다 무언가 망가지는 이슈

- **Docker의 본질**
  - **리눅스 컨테이너**를 일반 개발자도 쉽게 쓸 수 있게 만든 프로그램
  - `Dockerfile`에 OS·SDK·라이브러리·실행 코드를 명시 → 명령어 하나로 이미지 빌드 → 클릭 한 번으로 컨테이너 실행
  - 비유: **치킨집 밀키트**처럼 환경+코드를 통째로 포장해서 배포

- **Docker가 바꾼 개발 문화**
  - **MSA(Microservice Architecture)**: 모놀리식 → 작게 쪼갠 여러 프로그램으로 분리
  - **자동 배포 파이프라인** 도입 가속화
  - 로컬에 직접 설치하지 않고 컨테이너로만 돌리는 "결벽증" 개발자 등장

---

## 💡 개발자 포인트
- Docker의 실체는 리눅스의 가벼운 가상화 기술 위에 사용성 레이어를 얹은 것 — Docker 외에도 **대체 컨테이너 런타임**이 존재하며, 직접 리눅스 컨테이너를 띄우는 것도 가능
- **단점도 명확히 인지할 것**
  - 새로운 보안 이슈 관리 필요
  - 컨테이너 수가 늘면 운영 인력·서버 비용 증가
  - **영구 데이터 저장**에는 컨테이너가 적합하지 않을 수 있음 (볼륨/외부 스토리지 별도 고려)

> ⚠️ **입문자 주의**: 코딩 입문 단계에서 Docker부터 세팅하는 학습은 "남이 만든 밀키트만 데우는 요리 학습"과 같음. 먼저 로컬 환경에서 직접 부딪혀보는 경험이 있어야 나중에 `Dockerfile`을 자신 있게 작성하고 배포까지 다룰 수 있음.

- 후속 학습 추천: **Docker Compose** 같은 오케스트레이션 툴까지 함께 익히면 멀티 컨테이너 환경을 효율적으로 관리 가능
