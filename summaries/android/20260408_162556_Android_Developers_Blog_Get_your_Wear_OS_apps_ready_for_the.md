# Android Developers Blog: Get your Wear OS apps ready for the 64-bit requirement

## 핵심 요약 (3줄)
- 2026년 9월 15일부터 Wear OS 앱도 네이티브 코드 포함 시 64비트 버전을 필수로 제공해야 함
- 대부분의 Wear OS 개발자는 이미 64비트 전환을 완료했으며, 미전환 앱의 경우 작업량은 크지 않을 것으로 예상
- Kotlin/Java만 사용하는 앱은 코드 변경 불필요하나, 서드파티 SDK/의존성에 네이티브 코드 포함 여부를 반드시 확인해야 함

## 주요 발표 내용
- **64비트 필수 요건 적용 대상 확장**: 기존 모바일(2019년), Google TV에 이어 Wear OS에도 64비트 요건 적용
- **2026년 9월 15일부터 강제 적용**:
  - 네이티브 코드를 포함한 모든 신규 앱 및 업데이트 앱은 64비트 버전 필수 제공
  - 미준수 앱은 Play Console 업로드 시 차단됨
- **32비트 지원 정책은 유지**: 기존 32비트 디바이스에는 계속 앱 배포 가능
- **6개월 유예 기간 제공**: 현재 공지 기준으로 9월까지 전환 준비 가능

## 개발자에게 중요한 포인트
- **APK 분석 필수**: Android Studio의 APK Analyzer로 `.so` 파일 포함 여부 확인
  - 32비트 라이브러리: `lib/armeabi-v7a`
  - 64비트 라이브러리: `lib/arm64-v8a`
- **ABI 대칭 원칙**: 지원하는 32비트 아키텍처마다 대응하는 64비트 `.so` 파일도 함께 제공해야 함
- **서드파티 SDK 점검**: 직접 네이티브 코드를 작성하지 않더라도, 사용 중인 라이브러리나 SDK가 32비트 전용 네이티브 코드를 포함할 수 있으므로 공급사에 64비트 버전 요청 필요
- **테스트 환경**: Wear OS Android Emulator로 64비트 환경 동작 검증 가능
- **주의할 네이티브 코드 로더**: `SoLoader`, 구버전 `OpenSSL` 등은 64비트 환경에서 문제가 생길 수 있으므로 업데이트 필요
- **Wear OS 4 이상 타겟팅 앱**: 이미 64비트 전용 이미지에서 테스트 중일 가능성이 높아 추가 작업이 최소화될 수 있음

## 출시 일정 / 버전 정보
- **2026년 9월 15일**: Wear OS 64비트 요건 강제 적용 시작 (신규 앱 및 업데이트 앱 대상)
- **참고 기준 버전**: Wear OS 4 이상 (Google Play 제출 시 이미 필수 타겟)
